A Quick Guide to Get Started with JavaScript Classes

What is a class?

  • Classes were added to JavaScript in ES6 specification (ECMAScript 2015).
  • Classes are not a new feature per se. Classes basically provide a different way to create objects and work with prototypes and inheritance. This is also why many JavaScript developers call classes a syntactic sugar.
  • Every class starts with 'class' keyword
  • The convention is to write a name of the class startting with a capital letter
  • While declaring a class no parentheses and parameters are required.

Classes and Constructor

  • Classes don't support parameters, but still you can specify parameters for your class in the method called constructor.
  • Constructor is a unique method that is created inside a class and only once.
  • The constructor executes tasks you have specified when you create a new instance of a class.
  • Instance is a new object based on a specific class, and it inherits all properties and methods defined in that class.
  • Every time you create new instance of a class it will also automatically invoke the constructor method.
  • If you specify some parameters for the constructor method these parameters will become parameters of the class itself.
  • While creating new instance of the class, you can pass in some values as arguments, based on the parameters of the constructor.
  • The 'this' keyword inside a class refers to the class itself.

Class Properties and Methods

  • A class can have infinite number of properties, just like any object.
  • You can define properties for the class and assign thier values only inside the constructor method.
  • Alternatively, you can specify class fields( same as properties) outside of constructor but inside of class.
  • Three types of class fields: public, static and private
  • Class methods are defined inside the class body but outside of constructor.
  • Defining class methods is as simple as a function. Only the difference is: a class method is defined without a 'function' and 'this' keywords.
  • However, if you want to reference some property or method of the class you are working with, you need 'this' keywrod.
  • To call the method on the new instance that you have created, use dot notation.

Public class fields and methods

  • 'Class properties' and 'public class field' are very similar.

  • Differences between 'Class properties' and 'public class field':

    1. Class properties are defined in the constructor method.
    2. Class fields are defined outside of constructor.
    3. But we still need to use constructor, if you want to define class parameters or do some stuff during class intantiation.
    4. The public fields don't use 'this' keyword. To define new public fields, we start with the name of the field(property), not the 'this' and dot notation.
    5. Public fields are always accessible from the inside as well as outside of the class, and its instances.
    6. That means, we can access and modify the public fields. And we can do the same for public methods as well.
    7. Any class field and method defined are public by default

Static class fields and methods

  • We define a static class field or method by adding the keyword 'static' befor the field or method name.
  • Unlike public class fields, we can't access static class fields on instances of the class.
  • We can access static class fields only on the class itself.
  • The same applies to static methods as well. We can't call them on instances of the class. But we call them only on the class itself.
  • Static fields and methods are often used for utility purposes. For example, doing cleanups, updates or having an evidence of existing class instances.
  • Only static methods can work with static class fields. We can't access static class fields with neither public nor private methods.

Private Class Fields and Methods

  • You can work with fields or methods defined as private only inside the class.
  • From the outside, private field and methods are invisible.
  • We start the name with #(hashtag symbol) in order to define private field or method.
  • This can be useful to keep some data private. Such data are inaccessible from the outside and also from any class instance.
  • To access private field or cal private method, you have to use the #hashtag symbol.
  • Public method can access private fields and methods.

Classes and Instances

  • Instances are like new objects you create based on existing classes. The reason for creating new instances is that they automatically inherit properties and methods you defined in the class they are based on.
  • Therefore, you don't have to write the same code over and over again if you want to use it in multiple objects. You just need to create one class and put the code you want to re-use there. When you need an object that can do all that stuff you can use that class to create new instance.
  • The instance will inherit properties and methods you defined in that 'parent' class.
  • In order to create new instance of class, declare a new variable and assign with 'new' keyword followed by the name of the class you want to instantiate and parentheses. If the class accepts any parameters, you pass them inside the parentheses that follow after the name of the class, otherwise leave parentheses empty.
  • This way you can create as many instances of a specific class as you want.
  • All properties and their values you 'hard-code' in the constructor of a specific class will be inherited by all instances of that class. Any properties you assign values passed as arguments will be dynamic. They will depend on arguments you use during instantiation.
