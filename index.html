A Quick Guide to Get Started with JavaScript Classes

What is a class?

  • Classes were added to JavaScript in ES6 specification (ECMAScript 2015).
  • Classes are not a new feature per se. Classes basically provide a different way to create objects and work with prototypes and inheritance. This is also why many JavaScript developers call classes a syntactic sugar.
  • Every class starts with 'class' keyword
  • The convention is to write a name of the class startting with a capital letter
  • While declaring a class no parentheses and parameters are required.

Classes and Constructor

  • Classes don't support parameters, but still you can specify parameters for your class in the method called constructor.
  • Constructor is a unique method that is created inside a class and only once.
  • The constructor executes tasks you have specified when you create a new instance of a class.
  • Instance is a new object based on a specific class, and it inherits all properties and methods defined in that class.
  • Every time you create new instance of a class it will also automatically invoke the constructor method.
  • If you specify some parameters for the constructor method these parameters will become parameters of the class itself.
  • While creating new instance of the class, you can pass in some values as arguments, based on the parameters of the constructor.
  • The 'this' keyword inside a class refers to the class itself.

Class Properties and Methods

  • A class can have infinite number of properties, just like any object.
  • You can define properties for the class and assign thier values only inside the constructor method.
  • Alternatively, you can specify class fields( same as properties) outside of constructor but inside of class.
  • Three types of class fields: public, static and private
  • Class methods are defined inside the class body but outside of constructor.
  • Defining class methods is as simple as a function. Only the difference is: a class method is defined without a 'function' and 'this' keywords.
  • However, if you want to reference some property or method of the class you are working with, you need 'this' keywrod.
  • To call the method on the new instance that you have created, use dot notation.
